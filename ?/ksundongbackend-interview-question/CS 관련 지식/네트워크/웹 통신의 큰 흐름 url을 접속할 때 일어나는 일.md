
## 답변
브라우저는 URL을 파싱해 HTTP Request Message를 운영체제에게 전송을 요청하고
도메인은 직접 사용할 수 없어 DNS Lookup을 진행합니다
브라우저 DNS -> hosts파일 -> OS DNS Cache를 수행하고도 찾지 못한다면 DNS 서버에 질의해 목적지 IP를 얻습니다.

요청한 메시지는 운영체제의 TCP/IP계층을 거치며
TCP-세그먼트, IP-패킷, Ethernet-프레임으로 캡슐화를 하며

이 프레임은 로컬네트워크(스위치,허브)을 지나 라우터에 도달하고 라우터는 ISP에 전달해 인터넷 백본을 통해 목적지 네트워크에 라우팅합니다.

목적지 네트워크의 방화벽이 트래픽을 확인하고 필요하면 캐시 서버를 이용해 응답을 할 수도 있습니다.
웹 서버까지 도달한 프레임은 OS 서버를 통해 역캡슐화를 통해 HTTP Request Message로 복원되고
웹 애플리케이션은 요청에 대한 응답을 생성해 다시 동일한 경로를 반대로 거쳐 클라이언트로 전달합니다.
