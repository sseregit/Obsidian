# 효율성 증대 명령어
`--permission-mode`

| 모드                | 설명                               | 적합한 상황                         | 안전성       |
| ----------------- | -------------------------------- | ------------------------------ | --------- |
| acceptEdits       | 파일 편집 작업은 자동 승인, 다른 도구 실행은 승인 필요 | 반복적인 리팩토링, 코드정리                | 비교적 안전    |
| plan              | 실행전 항상 계획(Plan) 제시 후 승인 받음       | 복잡한 기능 개발, 아키텍처 변경, 대규모 마이그레이션 | 매우 안전     |
| bypasspermissions | 모든 승인 절차 없이 즉시 실행                | 도구실행보다 다양한 권한이 반복적으로 요구        | 위험, 신중 필요 |

# 클로드 코드 전반적인 설정 변경
- `/config`
```vim
Auto-compact                              true
   Use todo list                             true
   Show tips                                 true
   Rewind code (checkpoints)                 true
   Verbose output                            false
   Theme                                     Dark mode
   Notifications                             Auto
   Output style                              default
   Editor mode                               normal
   Model                                     Default (recommended)
   Auto-connect to IDE (external terminal)   true
```
- Auto-compact
	- 콘텍스트 윈도우가 꽉 찼을 때 자동으로 `/compact`를 실행할지 여부
	- 작가 추천: true
- Use todo list
	- 클로드 코드가 긴 작업을 구현할 때 작업할 순서를 Todo 리스트로 만들고 사용자에게 보여줄지 여부
	- 작가 추천: true
- Show tips
	- Claude Code가 코딩 도중에 작업 힌트, 단축키 안내, 워크플로우 팁 등을 자동으로 화면에 띄워줄지의 여부
- Rewind code (checkpoints)
	- 체크포인트 기능을 켜고 끄는 옵션
- Verbose output
	- 출력 결과를 길게 보여줄지 여부
	- 작가 추천: false
		- false로 하더라도 상당히 많은 내용을 매번 읽게된다. 부족하면 true로
- Theme
	- 원하는 크롤드 코드 테마를 지정할 수 있는 옵션
- Notifications
	- 클로드 코드의 작업이 끝났을 때 어떤 알림을 보여줄지 선택 가능
	- 작가 추천: Terminal Bell
		- Kitty로 두고 쓸수 있는데 거슬리게 된다.
- Editor mode
	- normal mode와 vim mode중 선택가능
	- vim mode를 선택하면 vim 키바인딩으로 클로드 코드를 사용할 수 있음
	- 작가 추천: normal
		- 터미널 환경이다 보니 normal
		- vim을 이미 사용할줄 안다면 Vim 모드가 유용하지만 처음 사용한다면 난이도가 상당하다.
- Model
	- 사용할 모델을 선택하는 옵션
